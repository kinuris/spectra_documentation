Overview

This documentation provides a comprehensive reference for all URL endpoints in the Spectra Clothing management system. The system follows a modular approach with separate apps for different business domains.

All URLs in this system are web UI endpoints that return HTML responses. The system does not currently have a REST API, but instead uses traditional Django views to render templates.

Accounts Endpoints

The accounts app handles user authentication, user profiles, and user management in the system.

Login Page

GET
/accounts/login/

Displays the user login form. On POST, authenticates the user and redirects to the dashboard.

Form Parameters (POST):
Parameter Type Description
username string User's username
password string User's password

Logout

GET
/accounts/logout/

Logs out the currently authenticated user and redirects to the login page.

User Profile

GET
/accounts/profile/

Displays the profile information for the currently logged-in user. Also allows updating personal information.

Required authentication: Yes

User List

GET
/accounts/users/

Displays a list of all users in the system, with options to add, edit, or delete users.

Required role: Admin

Add User

POST
/accounts/users/add/

Form for creating a new user account with role assignment.

Required role: Admin

Edit User

POST
/accounts/users/{user_id}/edit/

Form for editing an existing user's information, including role assignment.

URL Parameters:
Parameter Type Description
user_id integer ID of the user to edit
Required role: Admin

Delete User

POST
/accounts/users/{user_id}/delete/

Endpoint for deleting a user account.

URL Parameters:
Parameter Type Description
user_id integer ID of the user to delete
Required role: Admin

Products Endpoints

The products app handles product management, including product details, categories, and images.

Product List

GET
/products/

Displays a list of all products with search and filter functionality.

Query Parameters:
Parameter Type Description
q string Optional. Search term to filter products by name
category integer Optional. Filter products by category ID

Add Product

POST
/products/add/

Form for creating a new product with its details, including category, supplier, pricing, and images.

Product Detail

GET

Displays detailed information about a specific product, including its images, category, and variants.

Parameter Type Description

URL Parameters:
Parameter Type Description

Get Product Variants (AJAX)

GET
/inventory/api/product-variants/

AJAX endpoint to fetch variants for a specific product. Returns JSON data.

Query Parameters:
Parameter Type Description
product_id integer Required. ID of the product to get variants for
Note: This is the only JSON endpoint in the current system. It's used for dynamic form updates.

Dashboard Endpoints

The dashboard app provides reporting, analytics, and system overview functionality.

Dashboard

GET
/dashboard/

Main dashboard view with summary statistics, recent orders, low stock alerts, and quick links.

Report List

GET
/dashboard/reports/

Lists all generated sales reports with options to view details or generate new reports.

Generate Report

POST
/dashboard/reports/generate/

Form for generating a new sales report by specifying type, date range, and other parameters.

GET
/dashboard/reports/{report_id}/

Displays detailed information for a specific sales report, including all metrics and top-selling products.

URL Parameters:
Parameter Type Description
report_id integer ID of the report to display

Delete Report

POST
/dashboard/reports/{report_id}/delete/

Endpoint for deleting a sales report.

URL Parameters:
Parameter Type Description
report_id integer ID of the report to delete

Analytics

GET
/dashboard/analytics/

Advanced analytics view with interactive charts and data visualization for sales, inventory, and customer metrics.

Query Parameters:
Parameter Type Description
period string Optional. Time period for analytics (daily, weekly, monthly, yearly)
start_date date Optional. Start date for custom date range
end_date date Optional. End date for custom date range
